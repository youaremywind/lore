Common setup issues usually fall into one boundary.

## Common boundaries

| Symptom | Check |
| --- | --- |
| server does not start | Docker logs, `DATABASE_URL`, PostgreSQL health, Redis URL |
| migration fails | database user permissions, `vector` extension availability, `pg_jieba` preload if used |
| `/api/health` fails | Web container logs and network port mapping |
| browser/API auth fails | `API_TOKEN`, bearer header, browser session, reverse proxy headers |
| setup cannot save model config | provider base URL, API key, model name, timeout |

For Docker installs, inspect logs from the directory containing `docker-compose.yml`:

```bash
docker compose ps
docker compose logs web
docker compose logs postgres
```
