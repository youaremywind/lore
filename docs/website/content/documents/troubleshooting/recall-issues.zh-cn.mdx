召回问题可能来自配置、内容或预期。

## 现象和处理

| 现象 | 可能原因 | 首要动作 |
| --- | --- | --- |
| 没有候选 | embeddings 或 views 未配置，记忆缺少 disclosure/glossary | 验证 setup，然后改善记忆 |
| 候选错误 | 记忆过宽或 priority 虚高 | 拆分宽泛笔记并降低 priority |
| 中文召回差 | PostgreSQL 回退到了 `simple` tokenizer | 安装 `pg_jieba` 或 `zhparser` |
| 召回慢 | 过多宽泛生成视图、模型/db 慢、timeout 太低 | 检查召回分析并调 limits |
| Agent 忽略召回 | hook 没运行，或 Agent 没打开节点 | 验证运行时 hooks 和使用记录 |

使用 Recall 页面比较 prompt、候选列表、分数，以及节点是否被打开。
