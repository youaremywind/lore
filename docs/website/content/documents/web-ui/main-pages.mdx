The Web Console is organized around the memory lifecycle.

## Main pages

| Page | Use it for |
| --- | --- |
| Memory | browse domains, inspect nodes, edit content, paths, disclosure, priority, and glossary |
| Recall | test queries, inspect candidates, scores, and usage state |
| Dream | run memory maintenance and review proposed changes |
| Settings | configure embedding, View LLM, recall behavior, cache, backup, and auth-related settings |
| Setup | complete first-run configuration after install |

Start with Memory when you want to understand what exists. Start with Recall when you want to debug whether the right memory appears for a real task.
