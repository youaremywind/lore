Dream 是 Lore 的记忆维护工作流。

用 Dream 审阅记忆质量、发现过期或重复笔记、检查召回模式，并准备改进建议。把它当作审阅助手，而不是无人监管的写入器。

## Dream 如何工作

Dream 读取记忆图谱和召回历史，识别维护机会：过期节点、重复内容、模糊的 disclosure trigger、噪声召回模式、缺失的 glossary 术语。它会以建议编辑的形式呈现，你可以逐条审阅并接受。

## Dream 适合做什么

| 任务 | 结果 |
| --- | --- |
| 找到过期或重复记忆 | 更干净的记忆图谱 |
| 发现过宽或模糊的披露触发条件 | 更好的未来召回 |
| 审阅噪声召回模式 | 更少无关候选 |
| 建议记忆编辑 | 更清晰的内容、路径、术语和触发条件 |
| 创建维护日记 | 可追溯的维护历史 |

在配置备份且重要记忆变更需要人工审阅时，Dream 最安全。还在调优 Lore 时，先手动运行。只有在备份、恢复和审阅习惯都稳定后，才安排定时运行。

## Dream 不应该做什么

不要把 Dream 当成无需审阅的大规模改写工具。凡是让记忆比来源材料更宽泛、更自信或更难追溯的变更，都应该拒绝。
