Web Console 是 Lore 的人工审阅界面。

当你需要有意识地查看或修改记忆，而不是依赖 Agent 会话时，使用它。你可以在控制台检查记忆路径、测试召回、配置模型 provider、审阅 Dream 建议、管理备份，并排查 Agent 连接问题。

## 什么应该放到 Web Console 里做

| 任务 | 什么时候使用控制台 |
| --- | --- |
| Memory review | 你想浏览路径、检查内容，并有意编辑一个节点 |
| Recall testing | 记忆存在，但没有在 Agent prompt 中出现 |
| Setup and settings | embedding、View LLM、召回阈值、缓存、Dream 或备份需要调整 |
| Dream review | 自动维护提出记忆变更，需要人工审阅 |
| Backup | 即将执行清理、迁移、Dream 或大范围编辑 |

控制台不是 Agent 召回的替代品。Agent 工作时仍使用 boot memory 和每次 prompt recall。当行为不清楚时，控制台是验证和修正记忆系统的地方。

## 日常规则

如果 Agent 的回答看起来使用了过期上下文，不要立刻把规则复制回本地 `AGENTS.md` 或 `CLAUDE.md`。先检查记忆、披露触发条件、术语和召回证据。
