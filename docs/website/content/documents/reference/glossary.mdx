## Core terms

| Term | Meaning |
| --- | --- |
| Boot memory | fixed startup context loaded from global and channel-specific paths |
| Recall | task-specific retrieval of candidate memories |
| Disclosure | the future situation where a memory should appear |
| Domain | top-level namespace such as `core`, `preferences`, or `project` |
| Memory path | stable URI for a memory |
| Glossary | aliases and keywords that help recall find the right memory |
| Priority | importance signal used alongside recall scoring |
| Dream | maintenance workflow that reviews and improves memory quality |
| View | generated searchable representation of memory content, such as a gist or question |
| Client type | runtime identifier such as `codex`, `claudecode`, `openclaw`, `pi`, or `hermes` |
