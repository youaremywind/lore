| 问题 | 回答 |
| --- | --- |
| Lore 是向量数据库吗？ | 不是。Lore 使用 PostgreSQL、pgvector、全文搜索、生成视图和显式记忆元数据来管理 Agent 记忆。 |
| 我必须使用 Docker 吗？ | 不必须。Docker 是最简单路径，但你也可以让服务器连接自己的 PostgreSQL，并手动配置客户端。 |
| 我需要 `pg_jieba` 吗？ | 中文召回推荐使用。Lore 可以回退到 `zhparser` 或 `simple`，但质量可能下降。 |
| 每个聊天结果都应该变成记忆吗？ | 不应该。只保存将来还应该生效的持久事实、偏好、规则、决策和工作流。 |
| `CLAUDE.md` 和 `AGENTS.md` 应该保留所有规则吗？ | 不应该。本地只保留启动 guidance，把共享持久规则移入 Lore。 |
| 多个 Agent 可以共享记忆吗？ | 可以。这正是主要目标：一个 Lore 服务器可以服务 Codex、Claude Code、Pi、OpenClaw、Hermes 和 MCP 客户端。 |
| 召回噪声很大怎么办？ | 先改善记忆结构：拆分宽泛笔记，明确 disclosure，调整 glossary，并降低虚高 priority。 |
