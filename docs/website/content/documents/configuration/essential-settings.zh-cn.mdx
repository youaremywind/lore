把这一页当作安装后的关键设置清单。Lore 要正常使用，需要两类设置：

| 类型 | 在哪里设置 | 例子 |
| --- | --- | --- |
| 部署设置 | 服务器环境变量或 Docker Compose `.env` | 数据库 URL、API token、Redis、存储路径 |
| 运行时设置 | `/setup`、`/settings` 或 `/api/settings` | embedding 模型、View LLM、召回、Dream、备份策略 |

实用规则：基础设施值在服务器启动前设置；模型和记忆行为通过 Web Console 管理。

## 必填服务器设置

| 设置 | 配置位置 | 作用 |
| --- | --- | --- |
| `DATABASE_URL` | Web 服务器环境变量 | 连接 Lore Web 和 PostgreSQL。没有它，服务器无法正常启动。 |

Docker Compose 部署通常指向自带的 `postgres` 服务；使用托管 PostgreSQL 时，改成你的数据库地址。

## 推荐服务器设置

| 设置 | 什么时候设置 | 说明 |
| --- | --- | --- |
| `API_TOKEN` | 服务器可能被不可信网络访问时 | Web UI、MCP 客户端和 hooks 都使用同一个 token。 |
| `REDIS_URL` | 多进程部署，或需要共享缓存时 | Lore 可以不用 Redis，但共享部署建议配置。 |
| `CACHE_KEY_PREFIX` | Redis 与其他应用共用时 | 避免缓存 key 冲突。 |
| `CACHE_LOCAL_MAX_ITEMS` | 不使用 Redis 时 | 限制进程内 fallback 缓存大小。 |
| `LORE_DB_POOL_SIZE` | 需要调整数据库连接数时 | 默认 `10`；不确定时保持默认。 |

Docker Compose 还会使用 `POSTGRES_DATA_DIR`、`REDIS_DATA_DIR`、`SNAPSHOT_DATA_DIR` 等宿主机路径，让数据在容器重启后仍然保留。

## 运行时模型设置

首次使用先在 `/setup` 完成，之后可以在 `/settings` 调整：

| 设置组 | 用于 | 建议 |
| --- | --- | --- |
| Embedding provider、base URL、API key、model | 语义召回和索引重建 | 使用 OpenAI-compatible embedding 端点。 |
| View LLM provider、base URL、API key、model | 生成视图和 Dream | 选择能稳定遵循结构化指令的模型。 |
| 召回设置 | 控制 Agent 在 recall block 中看到什么 | 先用默认值；有真实记忆内容后再调。 |
| Dream 设置 | 生成记忆维护建议 | 在信任流程前保持人工审阅。 |
| 备份设置 | 安全清理和迁移 | 大范围编辑或运行 Dream 前先配置。 |

## 快速定位

| 现象 | 先检查 |
| --- | --- |
| 服务器无法启动 | 在 `docker-compose.yml` 所在目录执行 `docker compose ps` 和 `docker compose logs web` |
| Web Console 正常但 MCP 失败 | `API_TOKEN`、MCP URL、`client_type`、Agent 环境变量 |
| 召回结果弱 | 记忆内容、disclosure、glossary，然后才是召回阈值 |
| Dream 建议质量差 | View LLM 设置和记忆质量 |
