Dream 和备份应该一起配置。

Dream 可以提升记忆质量，但备份才是让维护可回滚的基础。安排大范围维护前，先配置备份。

## 推荐顺序

1. 选择一个不会随容器重建而丢失的备份位置。
2. 手动运行一次备份。
3. 调整备份设置时，在测试环境验证恢复流程。
4. 在召回质量稳定之前，保持 Dream 审阅策略保守。
5. 启用定时运行前，先手动运行 Dream。
6. 批准记忆变更前，先审阅 Dream 建议。

## 需要检查的设置

| 区域 | 需要决定什么 |
| --- | --- |
| `dream.enabled` | 是否允许定时 Dream |
| `dream.cron` | 定时 Dream 的运行时间 |
| `dream.auto_approve_changes` | 除非你信任流程且已经验证备份，否则保持 false |
| `backup.enabled` | 是否允许定时备份 |
| `backup.retention_count` | 保留多少份备份 |
| `backup.webdav.*` | 是否需要异机备份 |

在拥有可信恢复路径之前，不要安排大范围自动维护。
