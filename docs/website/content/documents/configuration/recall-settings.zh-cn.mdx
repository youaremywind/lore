召回设置会改变 Agent 能看到多少候选，以及展示结果有多严格。

## 推荐设置

在记忆内容已经相对干净之后再调设置。

| 设置类型 | 影响 |
| --- | --- |
| display threshold | 隐藏低于指定分数的候选 |
| max display items | 限制 prompt 上下文中出现的候选数量 |
| query timeout | 避免慢检索阻塞 Agent prompt 流程 |
| match weights | 调整文本、术语、语义和优先级信号的权重 |
| exclude boot nodes | 避免固定启动上下文在 prompt 召回中重复出现 |

从保守设置开始：展示少量候选。当重要上下文缺失时，优先改善记忆结构。盲目提高数量会让 Agent 过度依赖弱相关记忆。

推荐起始值：display threshold `0.4`，max display items `3`。这些值能保持召回块足够短，Agent 会阅读它，同时仍能展示最相关的记忆。
