召回块是一组记忆线索。

```text
<recall session_id="sess-1" query_id="q1">
0.74 | preferences://user | concise replies · timezone
0.68 | project://my_app/deployment | cloudflare pages · build command
</recall>
```

分数是相关性信号，不是命令。依赖某条记忆之前，用 `lore_get_node` 打开它，并在工具要求时传入召回块中的 `session_id` 和 `query_id`。这样 Lore 才能记录召回的记忆是否真的被使用。

好的 Agent 行为：

- 行动前打开最相关的记忆；
- 跳过弱候选，而不是硬塞进回答；
- 如果召回块漏掉明显上下文，手动搜索；
- 如果任务改变了持久事实，结束后写入或更新记忆。
