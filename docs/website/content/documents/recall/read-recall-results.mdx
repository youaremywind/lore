Recall results explain what Lore found and why it might matter.

## Fields to read first

Look at these fields first:

| Field | Meaning |
| --- | --- |
| score | combined relevance score for display and ranking |
| URI | stable memory address to open before use |
| cue words | glossary, path, or content terms that matched the prompt |
| query/session IDs | identifiers used to record recall adoption |

In the Web Console recall analysis page, compare the prompt, candidates, scores, and opened/used state. A good result is not just a high score. It is a result the agent opened and used correctly.

If a useful memory appears but ranks too low, improve its disclosure trigger and glossary terms. If noisy memories appear often, split broad memories and lower priority on background notes.
