先改善记忆结构，再通过调阈值改善召回。

按这个顺序：

1. 让记忆聚焦：一个节点一个主题。
2. 给它具体路径。
3. 写清楚 disclosure 触发条件。
4. 添加用户和 Agent 实际会输入的术语。
5. 诚实设置优先级。
6. 确认 embedding 和 View LLM 已配置。
7. 内容干净后再调整召回设置。

常见问题：

| 现象 | 首选修复 |
| --- | --- |
| 有用记忆从不出现 | 添加 disclosure 和 glossary |
| 宽泛结果太多 | 拆分大记忆并降低优先级 |
| 中文查询很弱 | 验证 `pg_jieba` 或 `zhparser`；否则 Lore 会回退到 `simple` tokenizer |
| 缺少语义召回 | 配置 embedding provider 并重建生成视图 |
| 结果看起来过期 | 重大编辑后运行 Dream 或刷新生成记忆视图 |
