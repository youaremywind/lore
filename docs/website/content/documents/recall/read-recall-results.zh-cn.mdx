召回结果解释 Lore 找到了什么，以及为什么它可能重要。

先看这些字段：

| 字段 | 含义 |
| --- | --- |
| score | 用于展示和排序的综合相关性分数 |
| URI | 使用前应该打开的稳定记忆地址 |
| cue words | 与 prompt 匹配的 glossary、path 或 content 词 |
| query/session IDs | 用来记录召回采用情况的标识符 |

在 Web Console 的召回分析页中，对比 prompt、候选、分数和 opened/used 状态。好结果不只是高分，而是 Agent 打开并正确使用了它。

如果有用记忆出现但排序太低，改善它的 disclosure 触发条件和 glossary。如果噪声记忆经常出现，拆分宽泛记忆，并降低背景笔记的 priority。
