在清理、迁移、运行 Dream 或大范围编辑前备份记忆。

## 备份流程

1. 打开 Web Console settings 页面。
2. 检查备份存储已配置且可访问。
3. 在大范围编辑、导入或 Dream 建议前手动运行备份。
4. 在项目笔记中记录备份时间和目标位置。
5. 修改备份设置后，在测试环境验证恢复。

以下操作前备份最有价值：

| 操作 | 为什么先备份 |
| --- | --- |
| 大范围清理 | 删除或移动的记忆可能难以重建 |
| 导入旧笔记 | 导入可能产生重复或噪声记忆 |
| Dream 维护 | 审阅期间建议应可回滚 |
| 服务器迁移 | 存储、认证和环境变更可能暴露配置错误 |

从未恢复过的备份只是猜测。在急需之前先测试恢复。

备份不能替代审阅。它只是让维护可回滚。对于重要项目，建议在 Dream 或大清理前做一次手动备份，而不是只依赖定时任务。
