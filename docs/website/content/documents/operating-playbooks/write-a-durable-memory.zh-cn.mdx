为一个未来使用场景写一条持久记忆。

有用记忆应该容易找到、容易信任，并且足够窄，不会污染无关 prompt。

## 示例

```text
URI: project://my_app/auth
Priority: 1
Disclosure: When working on authentication, login, refresh tokens, or session security for MyApp.
Glossary: MyApp auth, JWT, refresh token, login
Content: MyApp uses JWT access tokens with refresh-token rotation. Token refresh bugs must be checked against replay protection and server-side revocation state.
```

## 清单

1. 选择正确域：`core://`、`preferences://` 或 `project://`。
2. 路径保持短且具体。
3. 把内容写成应该跨会话保留的事实、规则、决策或工作流。
4. 添加指出未来任务的 disclosure 触发条件。
5. 为别名和可能出现在 prompt 中的词添加 glossary。
6. 根据真实重要性设置 priority。
7. 运行一次应该找到这条记忆的召回查询。

## 避免

- 保存 secrets、tokens、credentials 或 private keys；
- 把无关项目事实合成一条宽泛记忆；
- 把同一条规则复制进本地 `AGENTS.md`、`CLAUDE.md` 和 Lore；
- 所有记忆都使用 priority `0` 或 `1`；
- 把 disclosure 写成模糊标签，而不是未来触发条件。
