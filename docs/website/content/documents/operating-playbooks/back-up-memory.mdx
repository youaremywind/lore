Create a backup before heavy cleanup, migration, import, or Dream maintenance.

## Backup routine

Use this routine when the memory graph has become important enough that losing it would slow future work.

1. Open the Web Console settings page.
2. Check that backup storage is configured and reachable.
3. Run a manual backup before broad edits, imports, or Dream suggestions.
4. Record the backup time and destination in your project notes.
5. Restore from the backup in a test environment when you change backup settings.

Backups are most valuable before these actions:

| Action | Why back up first |
| --- | --- |
| Large cleanup | deleted or moved memories may be hard to reconstruct |
| Importing old notes | imports can create duplicate or noisy memories |
| Dream maintenance | suggestions should be reversible during review |
| Server migration | storage, auth, and environment changes can expose config mistakes |

A backup that has never been restored is only a guess. Test restore before you need it urgently.
