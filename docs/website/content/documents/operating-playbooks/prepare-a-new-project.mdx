Create project memory before the agent needs it.

## Starting set

Recommended starting set:

1. `project://project_name/overview` — what the project is and where it lives.
2. `project://project_name/commands` — install, dev, test, build, and deploy commands.
3. `project://project_name/constraints` — architecture rules, forbidden changes, product constraints.
4. `project://project_name/deployment` — hosting, env vars, release process, rollback notes.
5. `project://project_name/glossary` or glossary terms on each memory — product names, aliases, acronyms.

After writing the first memories, run a realistic task prompt through recall. If the project overview does not appear, improve its disclosure and glossary.
