在 Lore 已安装、`/setup` 已完成，并且至少一个 Agent 运行时可以收到启动上下文之后，再执行这一步。

`CLAUDE.md`、`AGENTS.md`、插件规则和 hook guidance 这类本地文件适合作为启动引导文件。它们不应该长期充当共享用户偏好、项目规则、编码标准、部署命令或操作习惯的唯一事实来源。如果同一条指令应该影响多个 Agent，就把它迁移到 Lore。

## 本地应该保留什么

本地只保留帮助运行时加载 Lore 的少量启动文本：

- 连接到 Lore MCP 服务器或插件；
- 会话开始时加载启动记忆；
- 依赖记忆前先使用召回；
- 把被召回节点当作事实前先打开；
- 把持久变更写回 Lore。

不要在每个本地 Agent 文件里重复产品规则、用户偏好、项目部署命令或长流程文档。那会产生漂移：一个 Agent 收到新规则，另一个还保留旧规则。

## 不同类型指令应该迁移到哪里

| 来源材料 | 迁移到 |
| --- | --- |
| 每个 Agent 都应该遵守的规则 | `core://agent` |
| 稳定用户偏好、写作风格、协作风格 | `preferences://user` |
| 项目命令、约束、架构决策、部署说明 | `project://<project_slug>/...` |
| 仅 Claude Code 需要的行为 | `core://agent/claudecode` |
| 仅 Codex 需要的行为 | `core://agent/codex` |
| 仅 Pi 需要的行为 | `core://agent/pi` |
| 仅 OpenClaw 需要的行为 | `core://agent/openclaw` |
| 仅 Hermes 需要的行为 | `core://agent/hermes` |

这与 Lore 的启动机制一致：运行时启动会加载全局启动节点 `core://agent`、`core://soul`、`preferences://user`，以及当前客户端专属节点，例如 `core://agent/codex` 或 `core://agent/claudecode`。

## 先审计哪些文件

常见本地来源：

| 运行时 | 典型需要检查的本地文件 |
| --- | --- |
| Claude Code | `~/.claude/CLAUDE.md`、项目 `CLAUDE.md`、导入的 Claude guidance 文件 |
| Codex | `~/.codex/AGENTS.md`、项目 `AGENTS.md`、仓库级 Agent 指令 |
| OpenClaw | 插件 guidance、`~/.openclaw/openclaw.json` 插件配置、项目笔记 |
| Pi | `~/.pi/agent/extensions` 或 `PI_CODING_AGENT_DIR` 下的扩展 guidance |
| Hermes | `hermes-plugin/lore_memory/AGENT_RULES.md` 或本地 Hermes 插件 guidance |

使用当前机器上真实的运行时路径。不要假设另一个项目目录里的随机 `AGENTS.md` 就是 Codex 当前正在使用的文件。

## 迁移流程

1. 打开本地文件，把内容拆成共享、用户偏好、项目专属、运行时专属和临时内容。
2. 把共享规则迁移到 `core://agent`。
3. 把稳定个人偏好迁移到 `preferences://user`。
4. 把项目材料迁移到 `project://<project_slug>/...` 节点，标题使用 `commands`、`deployment`、`architecture`、`constraints` 等清晰名称。
5. 把运行时专属差异迁移到 `core://agent/<client_type>`。
6. 从本地文件中删除重复长内容。
7. 留下一段短启动说明，让 Agent 使用 Lore 和召回。
8. 重启运行时，让启动 hooks 重新加载启动基线。

## 拆分示例

不好的本地文件形态：

```text
CLAUDE.md and AGENTS.md both repeat:
- user communication style
- frontend standards
- project install/build/deploy commands
- Codex-only hook notes
- Claude-only plugin notes
```

更好的 Lore 形态：

```text
core://agent                         shared agent workflow and quality rules
preferences://user                   stable user preferences and working style
project://lorehub/commands           install, dev, typecheck, build commands
project://lorehub/deployment         deploy notes and release process
core://agent/codex                   Codex-specific hook or plugin behavior
core://agent/claudecode              Claude Code-specific behavior
```

本地 `AGENTS.md` 或 `CLAUDE.md` 变成很短：

```text
Use Lore as the durable memory source.
Load boot memory at startup.
Use recall for task-relevant context.
Open recalled nodes before relying on them.
Write durable new facts back to Lore.
```

## 验证迁移

删除本地重复内容后：

1. 重启 Agent 运行时。
2. 提一个应该需要已迁移规则的 prompt。
3. 确认启动上下文包含共享和运行时专属启动节点。
4. 当 prompt 提到项目时，确认召回能浮现项目专属记忆。
5. 如果召回漏掉记忆，改善记忆的披露触发条件和术语，而不是把规则复制回本地文件。

不要把 secrets、API keys、tokens 或一次性临时指令存进 Lore。Secrets 放在环境变量或部署密钥存储中；临时任务笔记留在当前聊天或 issue tracker 中。
