已连接的 Agent 会收到三类上下文。

| 上下文 | 来源 | 用途 |
| --- | --- | --- |
| Boot memory | `/api/bridge/startup` 或 `lore_boot` | 稳定基线：共享 Agent 规则、用户偏好和渠道专属规则 |
| Recall candidates | `/api/bridge/recall` | 带 session/query 标识的 prompt 专属记忆线索 |
| MCP tools | `/api/mcp?client_type=...` | 浏览、搜索、创建、更新、移动、删除并标记使用情况 |

Boot memory 不是 recall 的替代品。它是确定性的启动基线。Recall 是任务专属的，应该被当作候选线索阅读。

健康工作流：

1. 启动运行时并加载 boot context。
2. 为当前 prompt 收到 `<recall>` 块。
3. 依赖记忆前，用 `lore_get_node` 打开相关节点。
4. 完成工作。
5. 当变更应该影响未来会话时，把持久变更写回 Lore。
