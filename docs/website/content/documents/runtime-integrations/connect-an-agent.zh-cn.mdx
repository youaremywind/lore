安装器可以配置选中的 channel：

## 安装器路径

```bash
curl -fsSL https://raw.githubusercontent.com/FFatTiger/lore/main/scripts/install.sh | bash -s -- --channels codex,claudecode
```

服务器已经运行时，使用 `--base-url`：

```bash
curl -fsSL https://raw.githubusercontent.com/FFatTiger/lore/main/scripts/install.sh | bash -s -- --base-url http://127.0.0.1:18901 --api-token YOUR_TOKEN
```

共享连接配置位于 `~/.lore/config.json`；插件专属配置可覆盖它，旧环境变量只作为兜底：

| 值 | 含义 |
| --- | --- |
| `base_url` / `baseUrl` | Lore 服务器 origin，不带尾部斜杠 |
| `api_token` / `apiToken` | 服务器使用 `API_TOKEN` 时的 bearer token |
| `client_type` | 发送给 Lore 的运行时身份：`codex`、`claudecode`、`pi`、`openclaw` 或 `hermes` |
| startup hook | 为当前运行时加载启动记忆 |
| prompt hook | 为当前用户 prompt 获取召回候选 |
| MCP tools | 读取、搜索、创建、更新、移动和删除 Lore 记忆节点 |

修改插件文件、MCP 配置、hooks 或 token 后，重启 Agent。Codex 用户应该打开 `/hooks`，并在提示时信任 Lore hooks。

连接验证后，把重复的持久指令从本地 Agent 文档移入 Lore。本地只保留启动 guidance。
