Disclosure 描述一条记忆应该在什么时候出现。

好的 disclosure 很具体：

```text
When working on Cloudflare Pages deployment, custom domains, build commands, or Pages deploy hooks for LoreHub.
```

弱 disclosure 很模糊：

```text
Important project info.
```

## 面向未来 prompt 编写

Disclosure 应该描述触发记忆的场景，而不是概括记忆本身。包含可能的任务、症状、产品名和决策领域。

使用这个模式：

```text
When [doing task], [touching area], [debugging symptom], or [making decision] for [project/product].
```

## Disclosure 与 glossary 的区别

| 字段 | 用途 |
| --- | --- |
| Disclosure | 记忆什么时候应该展示 |
| Glossary | 别名、缩写、产品名、文件名和可能的搜索词 |
| Content | 实际持久事实、规则、决策或工作流 |

不要把每个关键词都塞进 disclosure。别名和替代名称应该放进 glossary terms。
