Disclosure says when a memory should appear.

A good disclosure is concrete:

```text
When working on Cloudflare Pages deployment, custom domains, build commands, or Pages deploy hooks for LoreHub.
```

A weak disclosure is vague:

```text
Important project info.
```

## Write for future prompts

Disclosure should describe the situation that should trigger the memory, not summarize the memory itself. Include likely tasks, symptoms, product names, and decision areas.

Use this pattern:

```text
When [doing task], [touching area], [debugging symptom], or [making decision] for [project/product].
```

## Disclosure vs glossary

| Field | Use for |
| --- | --- |
| Disclosure | when the memory should be shown |
| Glossary | aliases, acronyms, product names, file names, and likely search words |
| Content | the actual durable fact, rule, decision, or workflow |

Do not stuff every keyword into disclosure. Put aliases and alternate names into glossary terms instead.
