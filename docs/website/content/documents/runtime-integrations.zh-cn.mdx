在服务器健康且 `/setup` 完成后连接 Agent。

## 支持的运行时

| 运行时 | 集成方式 | Agent 获得什么 |
| --- | --- | --- |
| Claude Code | MCP 工具、SessionStart hooks、`CLAUDE.md` guidance | Boot context、每次 prompt 召回、记忆读写工具 |
| Codex | MCP 工具、插件 hooks、技能注册 | Boot context、每次 prompt 召回、记忆管理 |
| Pi | 扩展 hooks | 启动 guidance、每次 prompt 召回 |
| OpenClaw | 插件工具 | Boot guidance、召回注入、健康检查 |
| Hermes | Python skill/provider | Boot、召回、记忆工具 |

所有渠道共享同一个 Lore 服务器和记忆图谱。通过 Claude Code 保存的规则对 Codex、Pi、OpenClaw 和 Hermes 都可见。

## 连接方式

安装器支持的 channel 是 `claudecode`、`codex`、`pi`、`openclaw` 和 `hermes`。你可以安装所有 channel，选择子集，跳过 Docker，或把本地 Agent 指向其他地方运行的 Lore 服务器。

Agent 集成有两个不同任务：

1. **连接** — MCP URL、`client_type`、bearer token、插件文件和 hooks。
2. **记忆归属** — 共享持久规则保存在 Lore；本地 `CLAUDE.md` 和 `AGENTS.md` 保持很小。

本节只说明连接行为。连接可用后，使用[把本地 Agent 文档迁移到 Lore](/zh-cn/documents/installation/move-agent-instructions-to-lore)，把重复的长期指令从本地 Agent 文件中移除。
