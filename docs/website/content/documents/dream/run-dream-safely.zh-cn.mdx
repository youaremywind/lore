还在调优 Lore 时，先手动运行 Dream。

Dream 会读取记忆和召回证据，然后提出维护工作。这可能很有用，但它仍然是在维护持久记忆。把每次运行都当作一次小迁移。

## 安全流程

1. 先备份记忆。
2. 从 Web Console 启动一次手动 Dream。
3. 先读日记摘要，再看单条变更。
4. 接受建议前，审阅拟议记忆变更。
5. 检查建议是否改善披露触发条件、术语、优先级或内容焦点。
6. 拒绝混合无关主题的大范围改写。
7. 接受变更后，运行真实召回查询验证行为。

## 什么时候避免运行 Dream

不要在以下情况把 Dream 当成自动大清理：

- embedding 或 View LLM 设置仍不稳定；
- 备份还没有验证；
- 刚导入了很多记忆且还没审阅；
- 召回质量差是因为缺少披露触发条件或术语。

只有在设置、备份和审阅习惯稳定后，定时 Dream 才有意义。
