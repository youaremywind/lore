像审阅代码变更一样审阅 Dream 建议。

## 审阅清单

| 检查项 | 接受条件 |
| --- | --- |
| scope | 一条建议只解决一个明确问题 |
| disclosure | 写清楚这条记忆应该在什么时候出现 |
| glossary | 术语很可能会出现在未来 prompt 中 |
| priority | 重要性没有被夸大 |
| safety | 没有编造事实，也没有抹掉有用约束 |
| traceability | 能理解这次编辑为什么有助于未来召回 |

拒绝让记忆比来源材料更宽泛、更模糊或更自信的变更。如果建议暴露了真实问题但改得太多，手动改写成更小的记忆。
