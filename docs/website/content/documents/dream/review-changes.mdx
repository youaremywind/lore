Review Dream suggestions like code changes.

## Review checklist

| Check | Accept when |
| --- | --- |
| scope | one proposed change addresses one clear issue |
| disclosure | it says when the memory should appear |
| glossary | terms are likely to appear in future prompts |
| priority | importance is not inflated |
| safety | the change does not invent facts or erase useful constraints |
| traceability | you can understand why the edit helps future recall |

Reject changes that make a memory broader, more vague, or more confident than the source material supports. If the suggestion uncovers a real issue but edits too much, rewrite a smaller memory manually.
