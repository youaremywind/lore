Lore 用来保存将来还会影响 Agent 行为的信息。

## 适合保存的内容

| 类型 | 原因 |
| --- | --- |
| 用户偏好 | 影响跨工具的后续回答 |
| 项目决策 | 影响后续编码和审阅工作 |
| 命令和工作流 | Agent 需要反复用到相同操作 |
| 部署约束 | 错误代价高，上下文容易丢失 |
| Agent 操作规则 | 启动时应一致生效 |

## 不适合保存的内容

| 不要保存 | 原因 |
| --- | --- |
| 临时便签 | 产生召回噪声 |
| 未验证的猜测 | 变成错误的持久上下文 |
| 大量粘贴日志 | 总结结论即可 |
| 重复的本地 Agent 规则 | 在 Lore 中集中保存一份 |

好的判断标准是：这个信息是否应该在几天或几周后仍然影响 Agent 的回答？如果是，就把它写成持久记忆。
