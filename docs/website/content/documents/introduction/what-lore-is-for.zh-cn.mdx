Lore 用来保存将来还会影响 Agent 行为的信息。

适合保存：

| 类型 | 示例 |
| --- | --- |
| 用户偏好 | 回复风格、语言、工作节奏、长期协作规则 |
| 项目知识 | 仓库位置、命令、部署流程、架构约束 |
| 决策 | 为什么选择某个方案，哪些替代方案被排除 |
| 工作流 | 常用检查清单、发布步骤、审阅标准 |
| Agent 规则 | 多个运行时都应该遵守的通用行为 |

不适合保存：一次性任务备注、临时草稿、秘密凭据、还没验证的猜测，或只对当前聊天有用的上下文。

好的判断标准是：这个信息是否应该在几天或几周后仍然影响 Agent 的回答？如果是，就把它写成持久记忆。
